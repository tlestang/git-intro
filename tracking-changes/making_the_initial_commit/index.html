Recording the initial version

The output of git status displays information about the state of your project.

git status
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	example_data.csv
	plot_data.py

nothing added to commit but untracked files present (use "git add" to track)

Git does not automatically track files in the project. To instruct Git to start tracking changes to a particular set of file, use the git add command.

git add plot_data.py example_data.csv

You can now record the state of the files, i.e. take your first snapshot of the project’s state. In Git, the action of making a record in the database of version is refered to as committing the changes. This is done with the git commit command:

git commit
[main (root-commit) 797ac1e] Add Python script
 2 files changed, 13 insertions(+)
 create mode 100644 example_data.csv
 create mode 100644 plot_data.py
